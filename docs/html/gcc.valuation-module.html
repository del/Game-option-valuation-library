value_gcc(S, X, Y, r, T, **params)

source code 

Values a GCC.

Parameters:
  • S ((L+1) x N-array) - the simulated underlying paths. L is the number of time steps - 1 (timesteps are numbered 0, 1, ..., L), and N is the number of simulated paths,
  • X ((L+1) x N-array) - the payoffs to the option holder when the writer terminates,
  • Y ((L+1) x N-array) - the payoffs to the option holder when he exercises,
  • r (number) - the risk-free interest rate,
  • T (number) - the maturity time, measured in years,
  • params - optional parameters,
  • m (integer) - the number of dimensions in the projection subspace when using the LSE method of valuation,
  • proj_type (string) - the type of functions in the projection subspace, as recognised by gcc.polynomials; e.g. "hermite" or "laguerre".
Returns:
a dict object containing all the input parameters, as well as:
  • V, the option price,
  • var, the Monte-Carlo variance
  • dev, the square root of var,
  • L, the number of time steps - 1,
  • dt, the size of a timestep, equal to T/L
  • time, the running time of the option pricing.
Notes:
  • When m is set, the LSE method will be employed, otherwise not.
  • Any further parameters will be ignored, but emitted into the output, so they can be used to annotate the output.
  • The parameters S, X, and Y must all be NumPy arrays with their shapes properly set. r and T will be cast to float64.

exp_holding_value_lse(S, X, Y, sigma, tau, j, lse_opts)

source code 

Calculate the expected holding value E[R(sigma_{j+1}, tau_{j+1})|j] by projecting R(sigma_{j+1}, tau_{j+1}) onto an m-dimensional subspace of F_j-measurable polynomial functions.

Parameters:
  • S - the simulated underlying paths. L is the number of time steps - 1 (timesteps are numbered 0, 1, ..., L), and N is the number of simulated paths,
  • X ((L+1) x N-array) - the payoffs to the option holder when the writer terminates,
  • Y ((L+1) x N-array) - the payoffs to the option holder when he exercises,
  • sigma ((L+1) x N-array) - the optimal stopping strategy for the writer of the option,
  • tau ((L+1) x N-array) - the optimal stopping strategy for the holder of the option,
  • j (integer) - the time step to evaluate the payoff at,
  • lse_opts (dict,) - a dictionary of options for the LSE,
  • m (integer) - the number of dimensions in the projection subspace when using the LSE method of valuation,
  • type (string) - the type of functions in the projection subspace, as recognised by gcc.polynomials; e.g. "hermite" or "laguerre".
Returns:
an N-array containing the expected holding values.

calculate_optimal_stopping_times(S, X, Y, lse_opts={})

source code 

Calculates optimal stopping strategies for the payoff R(sigma, tau) = X_sigma*I(sigma < tau) + Y_tau*I(tau <= sigma).

Parameters:
  • S - the simulated underlying paths. L is the number of time steps - 1 (timesteps are numbered 0, 1, ..., L), and N is the number of simulated paths,
  • X ((L+1) x N-array) - the payoffs to the option holder when the writer terminates,
  • Y ((L+1) x N-array) - the payoffs to the option holder when he exercises,
  • lse_opts (dict,) - a dictionary of options for the LSE, if it is to be used,
  • m (integer) - the number of dimensions in the projection subspace when using the LSE method of valuation,
  • type (string) - the type of functions in the projection subspace, as recognised by gcc.polynomials; e.g. "hermite" or "laguerre".
Returns:
L x N-arrays sigma and tau, containing the optimal stopping strategies from each time point. E.g., sigma[j, :] is the optimal stopping time restricted to {j+1, j+2, ..., L} for the writer of the option.

Note: When m is set, the LSE method will be employed, otherwise not.
